# V1.8 Enhancement Roadmap

本文定义 `multi-agent-company / one-person-company` 在 `V1.5` 封板之后、`V2` 之前的增强路线。  
这份文档只覆盖 `V1.8`，并明确以 `CEO 可见控制面` 为前提，不引入 `Company Plaza`。

## 1. 结论

`V1.8` 的目标不是扩部门，而是补强公司操作系统的四个中层能力：

- `Pulse / Trigger Engine`
- `Relationship Graph`
- `Skill Creator + Eval Loop`
- `CEO Visible Event Stream`

同时补入：

- memory promotion
- policy guard
- quotas

明确不做：

- `Company Plaza`
- `Growth / Support` 独立 bot
- `Slack`
- `L2 department promotion`
- `Quality` 双席位拆分

## 2. 四个正式模块

### 2.1 `Pulse / Trigger Engine`

目标：

- 让 recurring work、on-message automation、event-based routing 成为正式能力。

最小能力：

- `cron`
- `once`
- `interval`
- `poll`
- `on_message`
- `webhook`

不做什么：

- 不允许不可见 trigger 自治网络
- 不允许绕过 `Chief of Staff` 与审批链

依赖 `V1.5`：

- core-7 agent binding
- `RunTrace`
- `ConversationThread`
- `OpenClawSessionBinding`
- Feishu visible orchestration

### 2.2 `Relationship Graph`

目标：

- 正式定义谁可以接棒谁、谁可以召回谁、谁有收口权与默认升级路径。

最小能力：

- `default_handoff_targets`
- `allowed_repeat_recall_targets`
- `default_escalation_targets`
- `room-specific routing hints`

不做什么：

- 不做 agent 私下社交关系
- 不做不可见 peer graph

依赖 `V1.5`：

- visible handoff
- repeat recall
- `AgentDetailAggregator`
- room policy

### 2.3 `Skill Creator + Eval Loop`

目标：

- 把“自我进化”变成受控能力，而不是自由扩权。

最小能力：

- `skill proposal`
- `skill generation`
- `eval cases`
- `benchmark / score`
- `approval before install`

不做什么：

- 不做无审批自动装 skill
- 不做无评测的 skill promotion

依赖 `V1.5`：

- native skill export
- `SkillManifest`
- skill verification
- approval chain

### 2.4 `CEO Visible Event Stream`

目标：

- 用统一事件流替代 `Company Plaza`，把 agent 行为变成可审计、可追踪、可运营的控制面信息。

最小能力：

- handoff events
- trigger firing
- approval events
- memory promotion events
- skill invocation events
- room activity events

不做什么：

- 不做社交 feed
- 不做点赞评论式内部广场

依赖 `V1.5`：

- Dashboard `Agents` 模块
- `RunTrace`
- `Feishu Group Debug`
- skill audit
- room template audit

## 3. 治理层增强

`V1.8` 需要在四个模块之外补三类治理能力：

- `memory promotion`
  让经验从 `agent_private / department_shared` 升级到正式 procedural memory。
- `policy guard`
  统一 skill install、trigger firing、high-impact handoff 的边界。
- `quotas`
  对 skill、trigger、消息、自动化运行加入预算与频率约束。

这些能力都必须满足：

- 可审批
- 可审计
- 可回放
- 可停止

## 4. 与 V1.5 的关系

`V1.8` 不推翻 `V1.5`，而是建立在以下能力之上：

- Core-7 OpenClaw 对齐
- `AGENTS.md / SKILLS.md` bootstrap
- workspace-local native skills
- Dashboard `Agents` 模块
- visible handoff
- repeat recall
- Feishu card approval
- `Launch / Ops / Support Room`

如果以上任一能力未封板，`V1.8` 不应提前开工。

## 5. 进入 V1.8 的判定

只有当以下条件同时满足，才建议进入 `V1.8`：

- `V1.5 release checklist` 已签收
- Control UI 与 Dashboard 都只显示 core-7
- native skills 全量验证通过
- `Agents` 页面稳定
- Feishu live regression 全链路通过
- 没有新的未解释 dead-letter
